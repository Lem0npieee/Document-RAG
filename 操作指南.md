text


## 第2步：文档解析与多模态结构化分块

**核心思路**：不用本地OCR，也不用版面分析模型。直接用 `PyMuPDF` 把每页变成图片，然后让 `qwen3-vl-8b-instruct` 看图输出结构化数据。**在原有JSON输出的基础上，强制额外提取实体列表与实体关系三元组，为后续建图做原材料。**

**唯一指定动作：**

1. **写一个函数：PDF转图片**：使用 `pymupdf` 将上传PDF的每一页转为 `.png` 图片，保存到本地 `pages/` 目录，命名为 `page_1.png`，`page_2.png` 等。
    
2. **写一个函数：VL模型结构化提取（升级版）**：
    
    - 遍历每一张 `page_X.png`。
    - 调用 qwen3-vl-8b-instruct API，给出强制JSON输出的Prompt（**在原版基础上新增 `entities` 和 `relations` 两个字段**）：

text

```
你是一个文档解析专家。请分析这张文档图片，提取所有内容，并严格按以下JSON格式输出：
{
  "texts": ["第1段文字...", "第2段文字..."],
  "tables": ["表格1的Markdown格式...", "表格2的Markdown格式..."],
  "figures": [
    {"fig_id": "图1", "description": "详细描述图1展示的趋势或内容..."},
    {"fig_id": "图2", "description": "..."}
  ],
  "section_conclusions": ["第1节主要结论是...", "第2节主要结论是..."],
  "entities": [
    {"name": "ResNet", "type": "模型"},
    {"name": "图3", "type": "图表"},
    {"name": "准确率", "type": "指标"}
  ],
  "relations": [
    {"from": "图3", "relation": "展示了", "to": "准确率随训练轮次上升的趋势"},
    {"from": "ResNet", "relation": "在图3中表现最优", "to": "图3"},
    {"from": "第2节结论", "relation": "由数据支撑自", "to": "表格1"}
  ]
}
如果没有某项内容，对应值设为空数组。只输出JSON，不要输出其他内容。
```

- 将返回的JSON完整解析出来，`entities` 和 `relations` 单独存入一个列表，供第4步建图使用。

3. **打标签（构建元数据）**：把提取出的每一段文本、每一个Markdown表格、每一段图表描述，都封装成一个 LangChain `Document` 对象，并附加元数据：
    - 文本块：`{"source": "xxx.pdf", "page": 1, "type": "text", "node_id": "page1_text_0"}`
    - 表格块：`{"source": "xxx.pdf", "page": 1, "type": "table", "node_id": "page1_table_0"}`
    - 图表描述块：`{"source": "xxx.pdf", "page": 1, "type": "figure", "fig_id": "图3", "node_id": "page1_figure_3"}`

> **与原版相比的变化**：元数据中新增了 `node_id` 字段，这是图节点的唯一标识符，格式为 `页码_类型_序号`，贯穿后续所有模块。`entities` 和 `relations` 数据单独序列化保存为 `graph_data.json`，不入向量库。

---

## 第3步：向量化与FAISS入库

**核心思路**：把上一步提取的所有"文本/表格Markdown/图表描述文字"，全部用API转为向量，存入轻量级本地向量库。**此步骤与原版完全相同，不因加入GraphRAG而改变。**

**唯一指定动作：**

1. **初始化 Embedding 模型**：使用 `langchain_community.embeddings.DashScopeEmbeddings`（`text-embedding-v3`）。
    
2. **初始化 FAISS 向量库**：使用 `FAISS.from_documents(documents, embedding_model)`，将第2步生成的所有 `Document` 对象存入 FAISS。每个向量条目携带 `node_id` 元数据，作为向量空间与图空间的**桥梁键**。
    
3. **保存索引**：调用 `faiss.save_local("faiss_index")`，将向量库持久化到本地硬盘。
    

---

## 第4步：构建知识图谱

**核心思路**：读取第2步产出的 `graph_data.json`，用 `networkx` 将所有节点（文本块/图表/表格）和实体关系三元组组装成一张有向图，持久化保存。这是本项目区别于纯向量RAG的核心数据结构。

**唯一指定动作：**

1. **初始化有向图**：`G = networkx.DiGraph()`。
    
2. **添加内容节点**：遍历所有 Document 对象，以 `node_id` 为节点标识，将元数据（`page`、`type`、`fig_id`、`text content`）全部存为节点属性：
    

Python

```
G.add_node(
    "page1_figure_3",
    page=1,
    type="figure",
    fig_id="图3",
    content="图3展示了ResNet在CIFAR-10上准确率随训练轮次上升的趋势..."
)
```

3. **添加文档结构边（固定规则，无需VLM）**：
    
    - **同页归属边**：同一页的所有节点之间添加 `"同页"` 边（无向），反映空间共现关系
    - **跨节引用边**：若某页的 `section_conclusions` 文字中出现了 `fig_id`（如"图3"），则添加 `"支撑结论"` 有向边：`图3节点 → 该结论节点`
4. **添加VLM抽取的语义关系边**：遍历 `graph_data.json` 中每页的 `relations` 列表，将每条三元组 `{"from": A, "relation": R, "to": B}` 转化为图中的一条有向边，边属性存储关系类型 `R`：
    

Python

```
# 示例：("图3") --[展示了]--> ("准确率上升趋势实体节点")
G.add_edge("page1_figure_3", "accuracy_trend_entity", relation="展示了")
```

5. **持久化保存图**：使用 `networkx.write_gpickle(G, "doc_graph/graph.pkl")` 将图序列化到本地 `doc_graph/` 目录。
    
6. **输出图统计**：打印节点数、边数、平均度数，确认图构建成功。

## 第5步：构建多模态RAG问答链（无图，对照组A）

**核心思路**：在加入图检索之前，先实现**不含图谱的多模态RAG**作为对照组A。用户提问 → 向量检索文字 → 找到对应页码 → 把"原页截图"和"检索到的文字"一起丢给VL模型回答。

**唯一指定动作：**

1. **检索器构建**：从 FAISS 加载向量库，创建 `retriever = faiss.as_retriever(search_kwargs={"k": 3})`（返回最相关的3个块）。
    
2. **组装上下文**：
    
    - 获取检索到的3个块的元数据（提取 `page` 和 `fig_id`）
    - 根据元数据，去本地 `pages/` 目录读取对应的 `page_X.png` 原图
    - 将检索到的文本内容拼接到 Prompt 中
3. **调用 VL 模型生成回答**：
    

text

```
你是文档智能助手。请根据提供的"检索到的文本"以及"文档页面原图"，回答用户问题。
要求：必须在回答末尾标注引用来源，格式为 [引用：第X页, 图Y/表格Z/文本]。
检索到的文本：
{retrieved_texts}
```

- 将 **Prompt** + **页面原图（Base64）** 一起传给 `qwen3-vl-8b-instruct` API
- 输出最终回答
- 此模块保存为 `rag_chain.py`

## 第6步：构建多模态GraphRAG问答链（核心系统）

**核心思路**：在第5步的"向量检索"基础上，叠加"图谱游走"，实现**向量检索（语义）+ 图遍历（结构）+ 视觉理解（原图）**三轨合一的终极问答链。

**唯一指定动作：**

1. **第一轨——向量检索（种子节点获取）**：与第5步相同，用 FAISS 检索器找到 top-3 相关文档块，获取其 `node_id`，作为图游走的**种子节点**。
    
2. **第二轨——图谱游走（结构扩展）**：
    
    - 加载 `doc_graph/graph.pkl`
    - 对每个种子节点，在图 `G` 中执行**1跳邻居扩展**：`neighbors = list(G.successors(seed_node_id)) + list(G.predecessors(seed_node_id))`
    - 将邻居节点中 `type` 为 `figure` 或 `table` 的节点优先纳入扩展集（视觉证据优先）
    - 去重合并，得到"种子节点 + 1跳邻居"的**扩展节点集**，上限为8个节点
3. **证据融合**：
    
    - 从扩展节点集中提取所有节点的 `content`（文本/表格/图表描述）并拼接为 `retrieved_texts`
    - 从扩展节点集中提取所有涉及的 `page` 号，去重后读取对应 `page_X.png` 原图列表
    - 将**边的 `relation` 属性**（如"展示了"、"支撑结论"）也拼入Prompt，让VLM感知实体间的显式关系
4. **调用 VL 模型生成回答**：
    

text

```
你是文档智能助手。请根据以下多模态证据回答用户问题。
证据包含：①检索文本 ②实体关系链 ③文档页面原图。
实体关系链（知识图谱路径）：
{graph_reasoning_path}
检索到的文本与描述：
{retrieved_texts}
要求：回答末尾必须标注 [引用：第X页, 图Y/表格Z/文本, 经由关系: R]。
```

- 将 **Prompt** + **多张页面原图** 传给 `qwen3-vl-8b-instruct` API
- 此模块保存为 `multimodal_graph_rag_chain.py`

## 第9步：用 Gradio 封装界面

**核心思路**：花最少时间写一个可演示的Web界面，统一接入核心系统（多模态GraphRAG）。

**唯一指定动作：**

1. 写一个 `app.py`，使用 `gradio` 库：
    
    - 界面左侧：文件上传组件（`gr.File(file_types=['.pdf', '.png', '.jpg'])`）
    - 界面右侧：聊天框（`gr.Chatbot`）与文本输入框（`gr.Textbox`）
    - 知识图谱：生成一个知识图谱的界面，能看到各个知识（节点）和他们的关联（边）
2. **绑定逻辑**：
    
    - 上传文件时，依次触发第2步、第3步、第4步（解析 → 建向量库 → 建图谱），显示"文档解析完成，知识图谱已构建（X节点，Y条关系），可以提问"
    - 输入问题点击发送时，根据单选按钮的选择，路由到对应的问答链模块，将回答（含引用标注）显示在聊天框中
3. 运行 `python app.py`，浏览器打开本地链接，截图作为作业展示。
