### 第1步：注册API与准备环境（ Day 1 ）

**唯一指定动作：**

1. **注册阿里云百炼平台（DashScope）**：获取 API-Key。我们将100%使用阿里云的 `Qwen2.5-VL` (视觉语言模型) 和 `text-embedding-v3` (文本向量化模型)。新用户通常有免费额度，且价格极低。
2. **安装纯CPU依赖库**：在本地电脑或云服务器（如Colab CPU版）创建虚拟环境，运行以下命令：
    
    Bash
    
    ```
    pip install langchain langchain-community dashscope faiss-cpu pymupdf gradio python-dotenv
    ```
    
3. **配置密钥**：在项目根目录创建 `.env` 文件，写入：
    
    text
    
    ```
    DASHSCOPE_API_KEY=sk-你的真实密钥
    ```

### 第2步：文档解析与多模态分块（ Day 2-3 ）

**核心思路**：不用本地OCR，也不用版面分析模型。直接用 `PyMuPDF` 把每页变成图片，然后让 `Qwen2.5-VL` 看图输出结构化数据。

**唯一指定动作：**

1. **写一个函数：PDF转图片**：使用 `pymupdf` 将上传PDF的每一页转为 `.png` 图片，保存到本地 `pages/` 目录，命名为 `page_1.png`, `page_2.png` 等。
2. **写一个函数：VL模型结构化提取**：
    - 遍历每一张 `page_X.png`。
    - 调用 Qwen2.5-VL API，给出强制JSON输出的Prompt：
        
        text
        
        ```
        你是一个文档解析专家。请分析这张文档图片，提取所有内容，并严格按以下JSON格式输出：
        {
          "texts": ["第1段文字...", "第2段文字..."],
          "tables": ["表格1的Markdown格式...", "表格2的Markdown格式..."],
          "figures": [{"fig_id": "图1", "description": "详细描述图1展示的趋势或内容..."}, {"fig_id": "图2", "description": "..."}],
          "section_conclusions": ["第1节主要结论是...", "第2节主要结论是..."]
        }
        如果没有某项内容，对应值设为空数组。只输出JSON，不要输出其他内容。
        ```
        
    - 将返回的JSON解析出来。
3. **打标签（构建元数据）**：把提取出的每一段文本、每一个Markdown表格、每一段图表描述，都封装成一个文档对象（LangChain中的 `Document`），并强行加上元数据：
    - 文本块：`{"source": "xxx.pdf", "page": 1, "type": "text"}`
    - 表格块：`{"source": "xxx.pdf", "page": 1, "type": "table"}`
    - 图表描述块：`{"source": "xxx.pdf", "page": 1, "type": "figure", "fig_id": "图3"}`

### 第3步：向量化与入库（ Day 4 ）

**核心思路**：把上一步提取的所有“文本/表格Markdown/图表描述文字”，全部用API转为向量，存入轻量级本地向量库。

**唯一指定动作：**

1. **初始化 Embedding 模型**：使用 `langchain_community.embeddings.DashScopeEmbeddings`。
2. **初始化 FAISS 向量库**：使用 `FAISS.from_documents(documents, embedding_model)`，将第2步生成的所有 `Document` 对象存入 FAISS。
3. **保存索引**：调用 `faiss.save_local("faiss_index")`，将向量库持久化到本地硬盘。

---

### 第4步：构建多模态检索问答链（ Day 5-6 ）

**核心思路**：用户提问 -> 检索文字 -> 找到对应页码 -> 把“原页截图”和“检索到的文字”一起丢给VL模型回答。

**唯一指定动作：**

1. **检索器构建**：从 FAISS 加载向量库，创建 `retriever = faiss.as_retriever(search_kwargs={"k": 3})`（返回最相关的3个块）。
2. **组装上下文**：
    - 获取检索到的3个块的元数据（提取 `page` 和 `fig_id`）。
    - 根据元数据，去本地 `pages/` 目录读取对应的 `page_X.png` 原图。
    - 将检索到的文本内容拼接到 Prompt 中。
3. **调用 VL 模型生成回答**：
    - 构造 Prompt：
        
        text
        
        ```
        你是文档智能助手。请根据提供的“检索到的文本”以及“文档页面原图”，回答用户问题。
        要求：必须在回答末尾标注引用来源，格式为 [引用：第X页, 图Y/表格Z/文本]。
        检索到的文本：
        {retrieved_texts}
        ```
        
    - 将 **Prompt** + **页面原图（图片URL或Base64）** 一起传给 `Qwen2.5-VL` API。
    - 输出最终回答。

### 第5步：构建纯文本RAG对照组（ Day 7 ）

**核心思路**：为了满足作业“对比纯文本RAG与多模态RAG效果差异”的要求，写一个极简的降级版。

**唯一指定动作：**

1. **纯文本提取**：用 `pymupdf` 直接调用 `page.get_text()` 提取PDF纯文字（忽略所有图片，遇到表格也会变成乱码或纯文本）。
2. **纯文本入库与检索**：用同样的 Embedding 将纯文本切块存入另一个 FAISS 索引。
3. **纯文本大模型回答**：检索到纯文本后，调用 **纯文本模型**（如 `Qwen2.5-7B` 或 `Qwen-Turbo`，不要用VL模型），回答相同的问题。

### 第6步：评测与对比报告（ Day 8-9 ）

**核心思路**：不跑大型数据集，直接从 ChartQA 或 DocVQA 下载 30 个带图表的样本，手动对比。

**唯一指定动作：**

1. **准备测试集**：去网上找 3 份包含大量图表的学术论文PDF或财报PDF。自己针对这些PDF编写 30 个问题（10个纯文本问题，10个图表问题如“图3表示什么趋势”，10个表格问题如“表格中最高值是多少”）。写成一个 Excel 或 JSON 文件。
2. **跑评测**：分别把 30 个问题喂给“多模态RAG”和“纯文本RAG”。
3. **记录结果**：
    - 人工判断回答是否正确。
    - 统计结果写成表格：
        
        |问题类型|纯文本RAG准确率|多模态RAG准确率|典型错误案例|
        |---|---|---|---|
        |图表类 (10题)|0% (找不到图)|90%|纯文本RAG回答“未找到图3信息”|
        |表格类 (10题)|30% (格式错乱)|80%|...|
        |文本类 (10题)|80%|80%|...|
        
4. **得出结论**：纯文本RAG在处理视觉元素时完全失效，多模态RAG通过图表描述与原图双重保障，效果碾压。

### 第7步：用 Gradio 封装界面（ Day 10 ）

**核心思路**：花最少的时间写一个看得过去的Web界面，满足“系统可...”的验收要求。

**唯一指定动作：**

1. 写一个 `app.py`，使用 `gradio` 库：
    - 界面左侧：一个文件上传组件（`gr.File(file_types=['.pdf', '.png', '.jpg'])`）。
    - 界面右侧：一个聊天框（`gr.Chatbot`）和一个文本输入框（`gr.Textbox`）。
2. **绑定逻辑**：
    - 上传文件时，触发“第2步+第3步”（解析并建库），显示“文档解析完成，可以提问”。
    - 输入问题点击发送时，触发“第4步”，将回答显示在聊天框中。
3. 运行 `python app.py`，浏览器打开本地链接，截图作为作业展示。

### 总结：你的代码文件结构应该是这样

text

```
my_doc_assistant/
├── .env                  # 存放 DashScope API Key
├── requirements.txt      # 依赖库列表
├── pages/                # 存放PDF转换出的页面图片
├── faiss_index/          # 存放向量数据库文件

├── baseline_text_rag.py  # 第5步：纯文本对照基线
```

按照这个路线，你不需要装CUDA，不需要下载几十G的模型，一台普通的轻薄本或者免费的 Google Colab CPU 环境就能在10天内完美结课。现在就可以从**第1步**开始动手了。
