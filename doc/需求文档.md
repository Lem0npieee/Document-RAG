# 需求文档：多模态 GraphRAG 文档问答系统

## 文档说明

本文档对照 `开题报告.md` 中的系统设计方案，逐项梳理需求的完成状态。状态标记：

- ✅ 已完成 — 功能可用，已通过验证
- ⚠️ 部分完成 — 基本可用但存在已知问题
- ❌ 未开始 — 尚未实现

---

## 一、系统架构需求

### 1.1 知识库构建链路

| 编号 | 需求 | 状态 | 说明 |
|------|------|:---:|------|
| KB-01 | PDF 页面渲染（PyMuPDF → PNG） | ✅ | `src/parsing/pipeline.py`，支持自定义缩放比例 |
| KB-02 | VLM 结构化抽取（文本/表格/图表/结论/关键词/关系） | ✅ | 调用 qwen-vl-plus，含 repair prompt + text-only 兜底 |
| KB-03 | FAISS 向量索引构建 | ✅ | `src/indexing/faiss_store.py`，支持 DashScope / 本地 embedding |
| KB-04 | networkx 异构文档图谱构建 | ✅ | `src/graph/builder.py`，含内容节点/关键词节点/七种边类型 |
| KB-05 | 一键入库脚本 | ✅ | `build_multimodal_graphrag.py`，支持增量入库 + 产物复用 |
| KB-06 | 文件 sha256 注册表防重复解析 | ✅ | build_meta.json 记录已入库文件 |
| KB-07 | 多文档并发入库 | ⚠️ | 支持多文件但无并发，逐个串行处理 |

### 1.2 在线问答链路

| 编号 | 需求 | 状态 | 说明 |
|------|------|:---:|------|
| QA-01 | FAISS 语义召回（top-k 种子节点） | ✅ | `_retrieve_docs()`，k=3，无 source_hint 时候选 up to 30 |
| QA-02 | 图谱邻域扩展（一跳 + 二跳桥接） | ✅ | `_expand_with_graph()`，max_nodes=18 |
| QA-03 | 社区 profile 全局补充 | ✅ | `_select_global_profiles()`，greedy modularity communities |
| QA-04 | 多模态 prompt 生成（文本 + 关系 + 社区 + 图片） | ✅ | `ask()` 构造融合 prompt |
| QA-05 | VLM 生成答案 | ✅ | `vl_client.answer_question()` |
| QA-06 | 答案返回页码 + 节点 ID + 关系路径 + 图片路径 | ✅ | `GraphRAGResult` 包含完整引用信息 |
| QA-07 | source_hint 限定检索范围 | ✅ | 支持按文档源过滤，防跨文档污染 |
| QA-08 | 消融模式：vector_only（关闭图谱） | ✅ | `--ablation vector_only` |
| QA-09 | 消融模式：graph_only（关闭向量检索） | ✅ | `--ablation graph_only` |
| QA-10 | 消融模式：no_image（关闭页面原图） | ❌ | 开题报告 §5.4 要求但未实现 |
| QA-11 | 问答超时控制与错误处理 | ❌ | 当前无超时机制，大请求直接挂起 |
| QA-12 | 长答案/短答案模式切换 | ✅ | `answer_style` 参数，short 模式限制输出长度和图片数 |
