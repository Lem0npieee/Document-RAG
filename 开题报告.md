# 文档理解与多模态检索问答系统

### ——基于 Qwen2.5-VL + Knowledge Graph + LangChain 的多模态GraphRAG智能文档助手

---

## 一、任务定义与应用背景

### 1.1 任务定义

本项目旨在构建一个**多模态图增强文档智能助手（Multimodal Graph-enhanced Document Assistant）**，支持用户上传 PDF、扫描文档、PPT 截图等多种文档形式，系统能够：

1. **自动解析文档结构**：识别并定位文本段落、表格、图表、公式等不同类型的内容区域，同时抽取内容中的实体与实体间的关系；
2. **结构化分块入库**：将解析结果以带有来源元数据（页码、图号、类型、节点ID）的向量块形式存入 FAISS 向量数据库；
3. **知识图谱构建**：将同一文档内跨页的实体关系组织成 `networkx` 有向图，捕捉"图3展示了""支撑结论""引用自"等文档内部的语义与结构关系；
4. **多轨检索问答**：接收用户自然语言提问，通过"向量语义检索（召回种子节点）+ 图谱1跳游走（扩展结构相关节点）"双轨检索聚合证据，结合原始页面图像，调用视觉语言模型生成标注引用路径的回答；
5. **4组系统对比**：系统性对比纯文本RAG、多模态RAG、文本GraphRAG、多模态GraphRAG，在四类问题上定量评估各组差异。

核心研究问题是：**在多模态文档问答场景下，知识图谱结构与视觉理解能力各自贡献了多少效果提升？两者叠加的多模态GraphRAG能否在多跳推理题上实现决定性的优势？**

### 1.2 应用背景

随着数字化浪潮加速，企业、科研机构积累了海量图文混排文档（财报、学术论文、技术手册、法律合同等）。文档视觉问答（DocVQA）面临多重挑战：一是文档本身多模态（文本、图像、表格混排）；二是文档内容存在丰富的跨页逻辑关系（"第3节的图5引用了第1节的数据"）；三是传统向量检索仅靠语义相似度，无法捕获这种结构化的、需要多步推理的关系。

检索增强生成（RAG）通过引入外部知识库增强了模型的事实性；而知识图谱增强的GraphRAG进一步捕捉了实体间的结构化关系，使系统能够处理多跳推理。在此基础上融入多模态理解，是当前文档AI领域最前沿的研究方向。

具体应用场景包括：

- **学术科研**：追踪论文中"图5的结论来自第3节哪个实验"等跨章节逻辑链条
- **金融分析**：从财报中提取"图3的营收趋势与表4的分产品数据之间的关联"
- **法律合规**：定位"附件二中的图表与正文第5条款的引用关系"
- **企业知识管理**：跨文档、跨图表的多跳知识检索

## 二、相关工作调研

### 2.1 代表性数据集

**① DocVQA（Mathew et al., WACV 2021）**

DocVQA 是面向文档图像的视觉问答数据集，包含在 12,000+ 张文档图像上定义的 50,000 个问题。该任务超越了单纯的OCR，需要理解文档传递的各类信息，包括手写/打印文本、非文字元素（标记、勾选框、图表）、版面结构（表单、表格）以及字体样式等。DocVQA 是本项目评测的核心基准之一。

**② ChartQA（Masry et al., ACL Findings 2022）**

图表在数据分析中极为普遍，人们常提出涉及逻辑运算和视觉特征的复杂推理问题。ChartQA 提供了 9,600 道人工编写问题与 23,100 道基于图表摘要生成的问题，是目前规模最大的图表问答基准之一。本项目将使用 ChartQA 子集评测系统在图表理解上的能力。

### 2.2 代表性模型与方法

**③ Qwen2.5-VL（Alibaba Qwen Team, 2025）**

Qwen2.5-VL 在理解文档和图表方面具有显著优势，支持对发票、表单、表格等扫描件进行结构化输出，并且无需任务特定微调即可作为视觉代理（visual agent）使用。其视觉编码器为经过重新设计的 ViT 架构，引入了 2D-RoPE 与窗口注意力机制，原生支持可变输入分辨率。本项目通过阿里云百炼（DashScope）API 调用该模型，作为文档解析与多模态生成的双重引擎。

**④ ColPali（Faysse et al., 2024）**

现代检索系统主要依赖从文档页面提取的文本信息进行索引，这一过程往往繁琐且脆弱，难以高效利用视觉线索。ColPali 是一个经过训练的视觉语言模型，能够从文档页面图像中生成高质量的多向量嵌入；结合晚交互（late interaction）匹配机制，ColPali 性能大幅超越现代文档检索管道。本项目的视觉优先检索设计参考了其核心思想。

**⑤ MMDocRAG（2025）**

MMDocRAG 研究指出：视觉检索器更擅长检索图像内容，文本检索器擅长文本内容，混合检索器能提供更均衡的解决方案。这一结论直接支持了本项目采用"文本向量检索 + 视觉原图二次理解"混合架构的设计决策。

**⑥ Microsoft GraphRAG（Edge et al., arXiv 2404.16130, 2024）**

GraphRAG 由微软研究院提出，用以解决 LLM 在处理私有结构化数据时缺乏理解实体关系能力的问题，通过图数据库对实体关系建模，使其能够处理复杂查询并提升准确性。GraphRAG 利用图的关系结构实现基于领域特定查询的信息检索，并已在生产级应用中验证了其对多跳推理的显著改善效果。本项目的图谱构建与游走检索策略直接继承自该工作。

**⑦ Query-Driven Multimodal GraphRAG（ACL 2025 Findings）**

当前 RAG 和 GraphRAG 受限于静态知识库和低效的多模态数据整合；为此，研究者提出了查询驱动多模态 GraphRAG 框架，动态构建适配查询语义的局部知识图谱，通过：① 从查询语义推导图模式以引导知识提取；② 多路径检索策略精准定位核心知识；③ 按需补充缺失的多模态信息。本项目的图游走检索设计与该框架的多路径检索策略一脉相承。

**⑧ MMGraphRAG（arXiv 2025）**

MMGraphRAG 在 DocBench 上取得了 76.8% 的整体准确率，远超 NaiveRAG（59.5%）和 GraphRAG（52.3%）；这一优势在多模态查询上尤为突出（88.7% 准确率），展现了强大的视觉-文本融合能力。该结果为本项目预测"多模态GraphRAG在多跳推理题上大幅领先其他三组"提供了强有力的实证依据。

### 2.3 关键技术横向对比

|维度|纯文本RAG|多模态RAG|文本GraphRAG|**多模态GraphRAG（本项目）**|
|---|---|---|---|---|
|文档解析|`get_text()` 纯文字|VLM结构化JSON|`get_text()` 纯文字|**VLM结构化JSON + 实体关系抽取**|
|检索机制|向量检索|向量检索|向量 + 图游走|**向量 + 图游走**|
|图表理解|❌|✅ VLM描述|❌|**✅ VLM描述 + 原图**|
|多跳推理|❌|❌|✅|**✅**|
|生成模型|Qwen-Turbo|Qwen2.5-VL|Qwen-Turbo|**Qwen2.5-VL**|

## 三、系统方案设计

### 3.1 整体架构

系统分为**离线索引链路**与**在线问答链路**两条主线：

text

```
【离线索引链路】
用户上传 PDF
    │
    ▼
[PyMuPDF] PDF → page_1.png ... page_N.png
    │
    ▼
[Qwen2.5-VL API] 逐页图像 → 结构化JSON
    │  {texts, tables, figures, section_conclusions,
    │   entities:[{name,type}], relations:[{from,relation,to}]}
    │
    ├──────────────────────────────────────┐
    ▼                                      ▼
构建 LangChain Document                 提取 entities + relations
(含 node_id 元数据)                     存入 graph_data.json
    │                                      │
    ▼                                      ▼
[DashScope text-embedding-v3]        [networkx.DiGraph()]
向量化                                添加内容节点 + 结构边
    │                                 + 语义关系边
    ▼                                      │
[FAISS 本地向量库]                         ▼
持久化 faiss_index/              持久化 doc_graph/graph.pkl

【在线问答链路】
用户输入自然语言问题
    │
    ▼
[FAISS 检索器] top-3 → 获取 node_id（种子节点）
    │
    ▼
[networkx 图游走] 种子节点 1跳邻居扩展
(优先视觉节点figure/table，上限8节点)
    │
    ▼
融合：检索文本 + 关系路径 + page图片列表
    │
    ▼
[Qwen2.5-VL API]
Prompt(检索文本 + 关系链 + 问题) + 多张页面原图
    │
    ▼
生成回答（含 [引用：第X页, 图Y, 经由关系:R] 标注）
    │
    ▼
[Gradio 界面] 展示（支持4组系统切换）
```

### 3.2 核心模块说明

**模块1：文档解析（parse_doc.py）**

使用 `PyMuPDF` 将 PDF 每页渲染为 PNG 图片（2x分辨率）。将每张图片以 Base64 编码传入 `Qwen2.5-VL` API，强制输出包含 `texts`、`tables`（Markdown格式）、`figures`（含描述）、`section_conclusions`、`entities`、`relations` 六个字段的 JSON。将前四类内容封装为携带 `node_id` 元数据的 LangChain `Document` 对象；将 `entities` 和 `relations` 单独序列化为 `graph_data.json`。

**模块2：向量化入库（build_vectorstore.py）**

使用 `DashScopeEmbeddings`（`text-embedding-v3`）将所有 Document 向量化，通过 `FAISS.from_documents()` 构建索引并持久化。每个向量条目携带 `node_id`，作为向量空间与图空间的桥梁键。

**模块3：知识图谱构建（build_graph.py）**——**新增**

用 `networkx` 初始化有向图，以 `node_id` 为节点标识添加所有内容节点及其属性；添加三类边：① 同页归属边（规则生成）；② 跨节引用边（基于 `fig_id` 文字匹配）；③ VLM抽取的语义关系边（来自 `graph_data.json`）。持久化为 `graph.pkl`。

**模块4：多模态RAG问答链（rag_chain.py）**

FAISS top-3检索 → 读取对应页图像 → 组装多模态Prompt → 调用 `Qwen2.5-VL` 生成带引用标注的回答。（对照组A）

**模块5：多模态GraphRAG问答链（multimodal_graph_rag_chain.py）**——**新增**

FAISS top-3检索获取种子节点 → `networkx` 1跳图游走扩展节点集 → 融合检索文本、关系路径、多页原图 → 组装三轨Prompt → 调用 `Qwen2.5-VL` 生成带关系路径标注的回答。（核心系统）

**模块6：纯文本RAG（baseline_text_rag.py）**

`page.get_text()` 纯文本提取 → 独立FAISS索引 → `Qwen-Turbo` 生成回答。（对照组B）

**模块7：文本GraphRAG（text_graph_rag_chain.py）**——**新增**

复用 `graph.pkl` + FAISS双轨检索 → 仅传文本（不传图片）→ `Qwen-Turbo` 生成回答。（对照组C，单独验证图结构的独立贡献）

**模块8：评测（evaluation.py）**

读取 `test_questions.json`（40题），自动调用4组系统，人工判分后生成对比表格。

**模块9：界面（app.py）**

基于 `Gradio` 构建 Web 界面，支持文件上传触发全流程（解析→向量化→建图），支持4组系统切换的聊天交互。

### 3.3 模型选择理由

|组件|选型|理由|
|---|---|---|
|视觉理解与文档解析|Qwen2.5-VL（DashScope API）|文档/图表理解领域顶级性能；支持结构化JSON输出；无需本地GPU|
|知识图谱|networkx（本地，纯Python）|零额外成本，轻量级，无需数据库服务；支持节点属性与有向边，满足文档图建模需求|
|文本嵌入|DashScope text-embedding-v3|与VLM同平台，延迟低，支持中英文，免费额度充足|
|向量数据库|FAISS（CPU版）|轻量级本地存储，`pip install faiss-cpu` 即用|
|纯文本对照LLM|Qwen-Turbo|价格极低，与实验中对照组无视觉输入的设定一致|
|文档渲染|PyMuPDF|纯CPU高速PDF转图片|

### 3.4 评测指标

|指标|描述|适用场景|
|---|---|---|
|**准确率（Accuracy）**|人工判断三分制（全对/部分对/错）|40题全部|
|**多跳命中率**|多跳推理题中，系统是否正确追溯了跨页引用链条|多跳推理类10题|
|**引用准确率**|回答中引用的页码/图号/关系路径是否正确|含视觉或图谱元素的题目|
|**ANLS**|平均归一化编辑相似度（DocVQA标准指标）|需精确答案的题目|
|**图结构独立贡献**|对照组C准确率 - 对照组B准确率|多跳推理类（拆解图谱贡献）|
|**视觉独立贡献**|对照组A准确率 - 对照组B准确率|图表/表格类（拆解视觉贡献）|

## 四、初步数据集与实验设想

### 4.1 评测数据集方案

**数据集A：公开学术论文 PDF（3份）**

- 选取3篇含有丰富图表（≥5幅图、≥3个表格、≥2个跨节引用关系）的中英文学术论文PDF
- 推荐来源：arXiv.org 计算机视觉/NLP方向近期论文

**数据集B：ChartQA 子集**

- 从 ChartQA（9,600道人工题）中抽取10道图表推理题用于评测

**数据集C：DocVQA 子集**

- 从 DocVQA（50,000道题）验证集中抽取10道结构化文档题

**自建测试集（`test_questions.json`）**：针对数据集A的3份PDF，人工编写40道测试题：

|问题类型|数量|示例|
|---|---|---|
|纯文本理解|10题|"第3节的主要结论是什么？"|
|图表理解|10题|"图4中哪个类别的准确率最高？"|
|表格查询|10题|"表2中模型A在哪个指标上得分最低？"|
|**多跳推理**|**10题**|**"图3引用的数据来自第几节的哪个实验？"**|

### 4.2 实验设计

**实验一：4组系统全题型横向对比**

|问题类型|纯文本RAG|多模态RAG|文本GraphRAG|**多模态GraphRAG**|
|---|---|---|---|---|
|图表类（10题）|~10%|~80%|~15%|**~90%**|
|表格类（10题）|~30%|~75%|~35%|**~82%**|
|文本类（10题）|~75%|~80%|~78%|**~82%**|
|**多跳推理（10题）**|**~5%**|**~30%**|**~65%**|**~88%**|

**实验二：消融实验——分离图结构与视觉信息的独立贡献**

|贡献来源|计算方式|预期结论|
|---|---|---|
|视觉信息独立贡献|对照A - 对照B（图表题）|图表题提升约 +70pp|
|图结构独立贡献|对照C - 对照B（多跳题）|多跳题提升约 +60pp|
|二者叠加效应|核心系统 - 对照B（多跳题）|叠加优于单独使用任一|

**实验三：引用标注质量评估**

对所有回答检查引用标注格式 `[引用：第X页, 图Y, 经由关系:R]` 是否正确，统计多模态GraphRAG在关系路径标注上的准确率。

### 4.3 算力预算

|资源|需求|方案|

## 五、预期创新点与贡献

1. **多模态文档知识图谱自动构建**：利用 VLM 直接从文档图像中抽取实体与关系，无需人工标注，实现文档内部"图文关联图谱"的零样本自动化构建；
2. **三轨融合问答**：向量检索（语义相关性）+ 图谱游走（结构多跳推理）+ 视觉原图（视觉感知），三轨证据融合，超越现有单一检索范式；
3. **消融实验解耦贡献**：通过4组对比实验的精心设计，可独立量化"图结构"与"视觉信息"各自的贡献，为后续研究提供参考；
4. **零GPU成本落地**：全流程基于 API 调用 + 轻量级本地库，在普通CPU机器上即可完整运行，大幅降低多模态GraphRAG系统的实现门槛；
5. **多跳推理能力**：通过图谱游走显式捕获跨页的"引用链"、"图文关联"等结构关系，在多跳推理题上相比纯向量RAG预期提升约 +60pp，填补现有文档RAG系统在结构推理上的空白。
