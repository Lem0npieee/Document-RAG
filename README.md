# Document-RAG

面向课程作业“文档理解 + 多模态检索问答系统”的方案型仓库，目标是实现一个基于 **qwen3-vl-8b-instruct + Knowledge Graph + LangChain** 的多模态 GraphRAG 文档智能助手，用于处理 PDF、扫描文档、PPT 截图等图文混排内容，并在回答中给出可追溯的页面/图表引用。

## 项目定位

这个项目不是通用产品仓库，而是围绕课程题目 3 的工程化实现：

- 输入：多页 PDF、文档截图、图表页
- 处理：文档解析、结构化分块、向量检索、知识图谱构建
- 输出：带引用来源的自然语言回答
- 范围：仅实现多模态 GraphRAG 主模型，不包含对比模型/对比试验

## 本阶段落地范围

按现有方案，本仓库本阶段计划实现的范围主要包括：

1. 自动解析文档结构，识别文本、表格、图表/图像等内容。
2. 将文档内容按节点和元数据进行分块，便于后续检索与追踪。
3. 基于文档内部实体关系构建知识图谱，用于补足单纯向量检索难以处理的跨页、多跳推理。
4. 在问答阶段同时利用文本证据、图像证据和图谱路径，生成带引用标注的回答。
5. 通过单一主模型完成演示闭环，不包含对照实验。

## 技术栈

当前方案中采用或拟采用的核心技术如下：

- `qwen3-vl-8b-instruct`：文档解析与多模态回答的核心模型
- `DashScope`：模型 API 调用入口
- `LangChain`：文档抽象与检索链路组织
- `FAISS`：本地向量索引
- `networkx`：知识图谱与图遍历
- `PyMuPDF`：PDF 渲染与页面图片生成
- `Gradio`：最终交互界面
- `python-dotenv`：环境变量配置

## 模块边界

下面按职责拆分当前方案中的核心模块。这里把“运行主链路”“基线系统”“评测与展示”分开，避免混在同一层。

| 模块 | 职责 |
| --- | --- |
| 文档解析 | 将 PDF 页面转为图片，并调用多模态模型提取文本、表格、图表描述、实体与关系 |
| 向量入库 | 将结构化内容封装为 `Document`，写入 FAISS，并保留 `node_id` 等元数据 |
| 图谱构建 | 将实体与关系组织为 `networkx.DiGraph`，保存跨页、跨内容的结构化连接 |
| 多模态问答链 | 使用向量检索召回相关页面，再结合页面原图生成回答 |
| 多模态 GraphRAG 问答链 | 在向量召回基础上叠加图谱游走，融合文本、图结构与视觉证据 |
| Web 界面 | 提供上传、处理、提问、结果展示的统一入口 |

## 阶段计划

按当前材料，推荐分四个阶段推进，每阶段都产出可检查的结果：

1. **阶段 1：工程骨架**
   - 产出：目录结构、依赖文件、配置约定、最小运行入口

2. **阶段 2：文档理解**

4. **阶段 4：问答与演示**
   - 产出：多模态问答接口、演示界面
   - 目标：完成可演示、可写入最终报告的闭环

## 运行步骤（只做多模态主模型）

1. 创建并激活虚拟环境（Windows PowerShell）：

```bash
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. 安装依赖：

```bash
pip install -r requirements.txt
```

3. 配置密钥：

- 复制 `.env.example` 为 `.env`
- 在 `.env` 中填写真实 `DASHSCOPE_API_KEY`

4. 构建知识库（输入PDF或图片）：

- 建议先把文档放到项目根目录下的 `doc/` 文件夹
- `--input` 既支持绝对路径，也支持仅传文件名（会自动在 `doc/` 下查找）

```bash
python build_multimodal_graphrag.py --input "你的文档.pdf"
```

5. 启动后端服务（含静态前端与API）：

```bash
python src/server.py
```

6. 浏览器打开：

- http://127.0.0.1:8000
